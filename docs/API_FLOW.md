# API Flow

## Brief Maestro Submission (`POST /api/sendBrief`)

### Flow

```
brief-maestro.html                  sendBrief.js                   Email
─────────────────                   ────────────                   ─────
submitContact()
  ↓
validateFields()
  ↓
generatePrompt()
  ↓
fetch(/api/sendBrief, POST)
  ↓                              edgeCheck() → 429 if rate limited
  ↓                              honeypotCheck()
  ↓                              emailDedup()
  ↓                              timingCheck()
  ↓                              registerLifecycle('queued')
  ↓                              queue.push(handler)
  ↓                              → Response: { requestId, queuePosition, success: true }
  ↓
  ← - - - - - - - - - - - - - -
  ↓
showSuccess() or retry(429)        ┌──────────────────────────────┐
                                   │ Queue Worker (async)         │
                                   │ registerLifecycle('processing')│
                                   │ → Generate PDF via PDFKit    │
                                   │ → Send admin email + PDF    │
                                   │ → Send client confirmation  │
                                   │ registerLifecycle('completed')│
                                   └──────────────────────────────┘
```

### Payload
```json
{
  "name": "string (required)",
  "email": "string (required, valid email)",
  "company": "string (optional)",
  "phone": "string (optional)",
  "prompt": "string (generated by generatePrompt())",
  "lang": "es|en",
  "formData": "object (all 14 sections)",
  "submittedAt": "number (Unix ms timestamp, required)"
}
```

### Response (202 — queued)
```json
{
  "success": true,
  "requestId": "uuid",
  "queued": true,
  "position": 0,
  "depth": 1
}
```

### Response (429)
```json
{ "success": false, "error": "RATE_LIMITED" }
```

### Retry Strategy (Client-side)
| Attempt | Delay | Total |
|---|---|---|
| 1 | 0ms | 0ms |
| 2 | 1000ms | 1000ms |
| 3 | 2000ms | 3000ms |
| 4 | 4000ms | 7000ms |

## Lifecycle States

| State | Meaning | Terminal |
|---|---|---|
| `queued` | In queue, waiting for worker | No |
| `processing` | Worker executing | No |
| `completed` | Both emails sent successfully | Yes |
| `failed` | Email failure after retries | Yes |
| `rejected` | Pre-queue failure (validation/rate-limit) | Yes |
