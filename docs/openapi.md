# OpenAPI Documentation

## OpenAPI 3.1 Specification Overview

The Platform API is documented using OpenAPI 3.1.0. The specification is auto-generated by the `OpenApiGenerator` and covers all public API endpoints.

### Base URL
```
https://api.platform.io/v1
```

### Specification Access
```bash
# JSON format
GET /api/v1/developer/openapi

# YAML format
GET /api/v1/developer/openapi?format=yaml
```

## API Versioning

The API uses version `4.5.0` as the current stable version. Versioning is handled via:
- URL path prefix: `/api/v1/`
- OpenAPI spec version field: `info.version`
- SDK package versions match API version

## Authentication

All endpoints require Bearer JWT authentication:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.platform.io/v1/plugins
```

Security scheme defined in OpenAPI spec:
```json
{
  "components": {
    "securitySchemes": {
      "bearerAuth": {
        "type": "http",
        "scheme": "bearer",
        "bearerFormat": "JWT"
      }
    }
  }
}
```

## Swagger UI and Redoc

The OpenAPI specification can be viewed interactively:

- **Swagger UI**: Available at `/docs/openapi.html` — interactive API explorer with Try It Out functionality
- **Redoc**: Available at `/docs/redoc.html` — clean, three-panel API reference documentation

Both tools consume the same OpenAPI 3.1 JSON specification generated at `/api/v1/developer/openapi`.

### Swagger UI Features
- Endpoint listing with request/response schemas
- Try It Out — send real API requests from the browser
- Authentication header configuration
- Schema explorer with model details

### Redoc Features
- Three-panel layout (endpoints, details, schemas)
- Search/filter endpoints
- Code samples in multiple languages
- Webhook and callback support
